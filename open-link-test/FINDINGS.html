E2E test findings — open-link-test hexes

Tested 2026-05-26 against all three Calliope mini hardware versions (v1, v2, v3). Python + bleak for raw probes; Chrome + @calliope-edu/mini-connection-widget@01efeb7 for widget e2e on mini v3.

Filename convention
The N in makecode-N-{ble,radio}.hex is the Calliope mini hardware version the hex targets: makecode-1-* → mini v1 (nRF51, legacy DAPLink); makecode-2-* → mini v2 (nRF52833, J-Link OB); makecode-3-* → mini v3 (nRF52833, DAPLink, hardware-identical to micro:bit V2).

Region readbacks — all twelve hashes match the static hex analysis

Every probed device reported region hashes matching the hex bytes byte-for-byte. The codal PartialFlashing service correctly exposes the PXT magic + hashes the editor compiled in.

Observations

partial_flashing=1 fix shipped After the pxt-calliope rebuild with partial_flashing=1 in pxt.json, all six MakeCode hexes now expose PFS over BLE with hashes matching the static hex analysis. The widget can partial-flash MakeCode programs on all three mini variants.
Mini v3 radio variants fully fixed python_radio and makecode-3-radio both enter pairing mode on A+B+Reset and expose the unbonded DFU char + PFS. Earlier broken state (DEVICE_BLE=0) is resolved in both editors.
Mini v1/v2 have no Nordic DFU Widget's BLE full-DFU path is mini-v3-only. For older hardware, the widget needs to route through partial-flash or fall back to USB.
BLE-on-app behaviour varies by target makecode-2-ble advertises BLE in app mode without A+B+Reset; makecode-1-ble stays BLE-off-in-app. The per-target pxt-calliope config differs.

How to reproduce

Plug in the relevant Calliope mini (DAPLink at E: for mini v3, legacy DAPLink or J-Link OB at D: for v1/v2), then:

cd open-link-test

# Static analysis on all 9 hexes
python scripts/analyze_all.py

# App-mode probe loop (adapt drive letter for legacy minis):
#   import probe, usb_flash
#   probe.MINI_DRIVE = 'D:/'
#   usb_flash.MINI_DRIVE = 'D:/'
python scripts/probe.py

# Pairing-mode probe of a single hex
python scripts/probe_paired.py flash <hex>
# A+B+Reset on the mini, hold for full grid fill
python scripts/probe_paired.py probe <key>

For widget e2e (mini v3 only):

cd ../flash-test/widget-demo
pnpm install              # one-time
pnpm dev                  # in one terminal
node puppeteer-driver.mjs --park   # in another; click connect
