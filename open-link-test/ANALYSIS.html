Filename convention
The N in makecode-N-{ble,radio}.hex is the Calliope mini hardware version the hex targets: makecode-1-* → mini v1, makecode-2-* → mini v2, makecode-3-* → mini v3 (nRF52833, hardware-identical to micro:bit V2). Only the mini-v3 hexes work on the v3 connected to this rig.

How codal decides partial-flash support

codal's MicroBitMemoryMap::findHashes() scans flash post-app for two magic patterns:

PXT magic

0x923B8E70  0x41A815C6
0xC96698C4  0x9751EE75

16-byte sentinel followed by 8-byte DAL hash + 8-byte MakeCode hash. Embedded by PXT linker when an editor compiles user code; used by pxt-calliope and codal blocks-runtime once Scratch blocks are bundled.

uPy magic

0x597F30FE …
0xC1B1D79D

16-byte header at the end of a page; region records sit before the header. Embedded by addlayouttable.py during the codal MicroPython build.

Neither magic present → REGION_INFO returns all-zero ranges → widget falls back to full DFU.

blocks.hex (no magic)

The codal blocks-runtime expects PXT magic + hashes to be appended by the editor when a user program is compiled and bundled. A bare blocks-runtime build has no magic, so PFS REGION_INFO returns all-zeros. By design: scratch-calliope flashes the runtime once over USB, then streams Scratch blocks over MbitMore at runtime rather than re-flashing.
